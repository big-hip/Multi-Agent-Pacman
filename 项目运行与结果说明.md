# CS188 Project 2 运行环境与使用说明

本文档用于说明本仓库（Multi-Agent-Pacman）的环境搭建、算法运行、结果查看与常见问题排查方法，适用于 Windows + Miniconda 场景。内容同时覆盖 Pacman 主任务与 `eight_puzzle_lab` 扩展实验。

## 1. 运行环境搭建

### 1.1 Python 版本

建议使用 Python 3.8+。本机已验证环境：

- Python: 3.13.12
- 解释器路径: `D:\software\miniconda\python.exe`
- `tkinter`: 可用（图形界面运行 Pacman 所需）

### 1.2 创建并激活 Conda 环境（推荐）

```powershell
conda create -n cs188_p2 python=3.10 -y
conda activate cs188_p2
```

说明：课程代码通常不依赖大型第三方库，使用纯 Python 标准库即可完成自动评分。

### 1.3 依赖库安装

本项目核心运行通常不需要额外 `pip install`。若你的环境缺少图形依赖，可按需安装/修复：

```powershell
# 一般无需执行，仅在报错时按需处理
conda install tk -y
```

### 1.4 环境自检

```powershell
python --version
python -c "import tkinter; print('tkinter ok')"
```

若输出 `tkinter ok`，说明图形界面运行条件满足。

## 2. 算法运行方法

### 2.1 运行入口

- 游戏主入口：`pacman.py`
- 自动评分入口：`autograder.py`
- 学生代码文件：`multiAgents.py`

### 2.2 常用命令

#### A. 运行可视化对局（便于观察策略）

```powershell
# ReflexAgent
python pacman.py -p ReflexAgent -l testClassic

# MinimaxAgent
python pacman.py -p MinimaxAgent -l minimaxClassic -a depth=2

# AlphaBetaAgent
python pacman.py -p AlphaBetaAgent -l smallClassic -a depth=2

# ExpectimaxAgent
python pacman.py -p ExpectimaxAgent -l smallClassic -a depth=2
```

#### B. 批量无图运行（便于统计）

```powershell
python pacman.py -p ReflexAgent -l testClassic -n 10 -q
```

#### C. 自动评分（推荐提交前执行）

```powershell
python autograder.py -q q1
python autograder.py -q q2
python autograder.py -q q3
python autograder.py -q q4
python autograder.py -q q5

# 全量评测
python autograder.py
```

### 2.3 参数说明

| 参数 | 含义 | 示例 |
|---|---|---|
| `-p` | 选择 Agent 类名 | `-p ReflexAgent` |
| `-l` | 选择地图布局 | `-l openClassic` |
| `-a` | 传递 Agent 参数 | `-a depth=2` |
| `-n` | 对局次数 | `-n 10` |
| `-q` | quiet 模式（精简输出） | `-q` |
| `-q qX` | autograder 指定题号 | `-q q3` |

## 3. 结果获取与查看方式

### 3.1 终端输出含义

运行后会在终端看到以下核心指标：

- `Score`: 单局得分
- `Average Score`: 多局平均分
- `Win Rate`: 胜率
- `PASS/FAIL`: 自动评分样例通过情况

### 3.2 输出路径

默认不自动写文件，结果直接打印在终端。若需留档，建议重定向到日志文件：

```powershell
python autograder.py -q q1 > results_q1.log
python autograder.py > results_all.log
```

### 3.3 数据记录建议

建议按题目记录以下字段，便于写实验报告：

| 字段 | 说明 |
|---|---|
| 运行日期 | 例如 `2026-04-10` |
| 提交版本 | Git commit id |
| 命令 | 完整运行指令 |
| 平均分 | `Average Score` |
| 胜率 | `Win Rate` |
| 通过情况 | `PASS/FAIL` |

## 4. 常见问题与排查

### 4.1 `python` 命令不可用或版本不对

现象：`python --version` 报错，或版本不是预期。

排查：

```powershell
where python
conda info --envs
conda activate cs188_p2
python --version
```

### 4.2 图形界面报错（`tkinter` 相关）

现象：运行 `pacman.py` 时出现 `No module named tkinter`。

处理：

```powershell
conda install tk -y
python -c "import tkinter; print('tk ok')"
```

### 4.3 自动评分卡住或运行慢

建议：

- 先单题运行（`-q q1` 到 `-q q5`）定位问题。
- 优先用 `-q` 无图模式做批量测试。
- 检查是否把搜索深度设得过大（如 `depth` 过高）。

### 4.4 Git 推送报 HTTPS 证书域名不匹配

典型报错：证书主体与 `github.com` 不一致。

原因：网络链路中代理/加速器做了 HTTPS 拦截。

处理建议：

1. 关闭代理/VPN/加速器后重试。
2. 改用 SSH 推送（推荐）。
3. 不建议关闭 SSL 校验（存在安全风险）。

SSH 方式示例：

```powershell
git remote set-url origin git@github.com:big-hip/Multi-Agent-Pacman.git
ssh -T git@github.com
git push
```

## 6. eight_puzzle_lab 扩展实验说明（补充）

`eight_puzzle_lab` 是仓库中的独立扩展实验目录，和 Pacman 自动评分互不冲突。该目录包含两类脚本：

- `run_experiment.py`：标准 8-puzzle 搜索算法对比（BFS/IDS/A*/Greedy）
- `adversarial_8puzzle_experiment.py`：对抗式扩展（Minimax/AlphaBeta/Expectimax）

### 6.1 运行命令

在仓库根目录执行：

```powershell
python eight_puzzle_lab/run_experiment.py
python eight_puzzle_lab/adversarial_8puzzle_experiment.py
```

### 6.2 输出路径与含义

两条脚本都会先在终端打印结果，并写出 Markdown 报告文件：

| 脚本 | 报告输出路径 | 结果指标 |
|---|---|---|
| `run_experiment.py` | `eight_puzzle_lab/experiment_report.md` | 是否求解、路径长度、扩展节点数、生成节点数、耗时、动作序列 |
| `adversarial_8puzzle_experiment.py` | `eight_puzzle_lab/8puzzle_adversarial_report.md` | 是否求解、步骤数、扩展节点数、耗时、求解者/环境动作轨迹 |

### 6.3 数据记录建议

建议额外记录以下字段，便于写补充报告：

- 测试用例名称（Case-A / Case-B / Case-C 或 Easy/Medium/Harder）
- 算法名称
- `solved`（是否求解）
- `expanded_nodes`（扩展节点）
- `elapsed_seconds`（时间）
- 路径或动作轨迹

### 6.4 常见问题排查（eight_puzzle_lab）

1. `FileNotFoundError` 或报告没生成：
	- 请从仓库根目录运行命令，不要在其他目录直接执行相对路径脚本。
2. 结果不一致：
	- 对抗实验中 `Expectimax` 含随机扰动，需固定随机种子或多次运行取平均。
3. 搜索时间过长：
	- 优先查看 `expanded_nodes`，必要时降低对抗实验深度参数（脚本内 `depth`）。
