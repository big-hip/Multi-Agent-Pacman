# CS188 Project 2 多智能体搜索作业完成报告

## 1. 项目背景

本项目是 UC Berkeley CS188 的 Project 2，主题为 Pacman 多智能体搜索。核心目标是让 Pacman 在存在多个幽灵智能体的对抗环境中做出更优决策，涵盖了以下经典 AI 方法：

- 反射式决策（Reflex Agent）
- 极小极大搜索（Minimax）
- Alpha-Beta 剪枝
- 期望极大搜索（Expectimax）
- 启发式状态评估函数设计

项目重点不只是“能运行”，而是要在统一评测框架下通过功能正确性、搜索深度语义、剪枝行为约束与实战表现等多维验证。

## 3. 补充实现与设计说明

本节说明在课程提供骨架代码基础上补充完成的关键实现。

### 3.1 Q1 Reflex Agent 评估函数改进

实现思路：

- 以 successor state 的原始分数为基础
- 对 STOP 动作增加惩罚，减少无效停留
- 使用到最近食物距离的倒数作为奖励，引导更积极吃豆
- 区分幽灵是否处于 scared 状态：
  - 幽灵可吃时，靠近幽灵给予奖励
  - 幽灵危险时，靠近幽灵给予惩罚
  - 与危险幽灵重合时直接返回负无穷，强制规避

效果上，该策略兼顾了“吃豆效率”和“生存安全”，满足 Q1 稳定通关要求。

### 3.2 Q2 Minimax

实现了多智能体版本的 Minimax：

- Pacman 作为 max 节点
- 每个幽灵作为 min 节点
- 通过 agentIndex 循环处理任意数量幽灵
- 使用 ply 语义控制深度：当所有智能体轮转回 Pacman 时 depth + 1
- 在胜/负局面、达到深度上限、无合法动作时执行评估函数回退

### 3.3 Q3 Alpha-Beta 剪枝

在 Q2 框架上补充 Alpha-Beta：

- 在 Pacman 节点维护 alpha 下界
- 在幽灵节点维护 beta 上界
- 使用严格不等比较进行剪枝（避免对相等值剪枝）
- 按合法动作原顺序展开后继，满足课程对评测展开顺序的要求

实现目标是保证结果与 Minimax 一致前提下减少无效搜索分支。

### 3.4 Q4 Expectimax

将幽灵建模为随机动作智能体：

- Pacman 仍取最大期望收益动作
- 幽灵节点不再取最小值，而是按均匀分布计算期望值
- 同样支持多幽灵与统一深度定义

该实现反映“对手并非总是最坏行动”的不确定环境决策逻辑。

### 3.5 Q5 Better Evaluation Function

实现了更强的状态评估函数，核心特征包括：

- 当前游戏分数（基础项）
- 最近食物距离奖励 + 剩余食物数量惩罚
- 最近能量豆距离奖励 + 剩余能量豆数量惩罚
- 幽灵风险/收益建模：
  - scared 幽灵靠近奖励
  - 非 scared 幽灵靠近惩罚
  - 与危险幽灵重合直接判为极差状态

该评估函数在进攻（吃豆、清图）与防守（避险）间取得较好平衡，满足 Q5 胜率与得分门槛。
