## 2. 算法运行方法

### 2.1 运行入口

- 主程序入口：pacman.py
- 自动评分入口：autograder.py
- 学生实现文件：multiAgents.py

### 2.2 常用命令

#### A. 可视化运行（观察策略行为）


| 参数 | 含义 | 示例 |

## 3. 结果获取与查看

### 3.1 终端输出重点

运行后重点关注：

- Score：单局得分
- Average Score：平均得分
- Win Rate：胜率
- PASS/FAIL：自动评分样例是否通过

### 3.2 结果留档

默认结果打印到终端；若需留档可重定向：

```powershell
python autograder.py -q q1 > result_q1.log
python autograder.py > result_all.log
```

### 3.3 建议记录字段

| 字段 | 说明 |
|---|---|
| 日期 | 运行时间 |
| 命令 | 具体执行指令 |
| 平均分 | Average Score |
| 胜率 | Win Rate |
| 结果 | PASS/FAIL |

## 4. 常见问题与排查

### 4.1 Python 命令不可用或环境错误

```powershell
where python
conda info --envs
conda activate cs188_p2
python --version
```

### 4.2 图形界面报 tkinter 错误

```powershell
conda install tk -y
python -c "import tkinter; print('tk ok')"
```

### 4.3 自动评分很慢

- 先按题分开跑（q1 到 q5），定位具体慢点。
- 大规模回归优先使用 -q 模式。
- 检查是否误把搜索深度设得过高。

### 4.4 Git 推送报 HTTPS 证书域名不匹配

现象示例：证书主体与 github.com 不一致。

原因：网络链路代理/加速器拦截 HTTPS，返回了非 GitHub 证书。

建议：

1. 关闭代理/VPN/加速器后重试。
2. 改用 SSH 推送（推荐）。
3. 不要关闭 SSL 校验。

SSH 参考：

```powershell
git remote set-url origin git@github.com:big-hip/Multi-Agent-Pacman.git
ssh -T git@github.com
git push
```
