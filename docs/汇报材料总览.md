# 汇报材料总览（CS188 Project 2）

## 1. 项目背景
本项目来自 UC Berkeley CS188 的 Project 2（Multi-Agent Search）。问题场景为 Pacman 与多个 Ghost 的多智能体决策：Pacman 需要在动态对抗环境中最大化收益并避免失败。该项目用于训练以下能力：

- 多智能体博弈搜索建模
- 不同对手模型下的策略设计
- 启发式评估函数设计与调参
- 通过自动评分框架进行算法正确性与性能验证

## 2. 我完成了哪些实现
代码实现集中在 [multiAgents.py](../multiAgents.py)。

### 2.1 Q1 Reflex Agent
位置：[multiAgents.py](../multiAgents.py#L55)

实现内容：
- 基于后继状态分数作为主干
- 引入最近食物距离奖励
- 对 STOP 动作惩罚
- 对 Ghost 区分 scared / 非 scared 两种策略
- 与危险 Ghost 距离为 0 时直接返回极小值

### 2.2 Q2 Minimax
位置：[multiAgents.py](../multiAgents.py#L130)

实现内容：
- Pacman 为 max 层，Ghost 为 min 层
- 支持任意数量 Ghost（通过 agentIndex 轮转）
- 深度以 ply 计：回到 Pacman 时才 depth + 1
- 终止条件：胜/负、深度到达上限、无合法动作

### 2.3 Q3 Alpha-Beta
位置：[multiAgents.py](../multiAgents.py#L195)

实现内容：
- 在 Minimax 框架中加入 alpha/beta 边界
- max 层更新 alpha，min 层更新 beta
- 利用剪枝减少展开节点
- 保持与 Minimax 一致的最优动作语义

### 2.4 Q4 Expectimax
位置：[multiAgents.py](../multiAgents.py#L251)

实现内容：
- Pacman 层仍做最大化
- Ghost 层按均匀概率计算期望值
- 建模随机对手，而非总是最坏对手

### 2.5 Q5 Better Evaluation Function
位置：[multiAgents.py](../multiAgents.py#L302)

实现内容：
- 融合当前分数、食物数量/距离、胶囊数量/距离
- Ghost 风险与收益分离建模
- 对危险接触给出强惩罚，对可食 Ghost 给出追击奖励

## 3. 关键技术与说明

### 3.1 为什么使用距离倒数而非原始距离
使用 $\frac{1}{d+1}$ 的好处是：

- 距离越近，分数变化越明显，更有利于区分“当前就能吃到”与“还很远”
- 距离很远时自动衰减，避免某单一特征主导全局

### 3.2 多智能体深度定义（统一语义）
本实现采用“完整一轮 agent 行动记为 1 层”的定义，即仅当轮回到 Pacman 才 depth + 1。这样可避免不同算法间深度语义不一致，保证 Q2/Q3/Q4 对比公平。

### 3.3 Alpha-Beta 的收益与边界
Alpha-Beta 通过边界裁剪无效分支，理论上不改变最优决策结果，但能减少节点展开，提升搜索效率。

### 3.4 Expectimax 的适用场景
当对手行为近似随机时，Expectimax 比 Minimax 更符合真实环境假设。它优化的是期望收益而非最坏收益。

### 3.5 评估函数的平衡策略
BetterEvaluationFunction 的核心不是单一追分，而是三目标平衡：

- 安全：远离危险 Ghost
- 效率：减少剩余食物与胶囊
- 收益：在 scared 窗口积极追击 Ghost
